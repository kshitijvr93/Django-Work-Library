• Compared to the SobekCM/UFDC metadata editor, the MAW metadata editor will incorporate much of the look and feel of the UFDC/SobekCM form-based metadata editor:
    • Form fields provide labels that are more descriptive than the typically shorter xml tags alone.
    • Fields with limited choices have drop-down lists to facilitate data entry.
    • Fields that are repeatable will also feature a 'Plus sign' button that allows the user to repeat the display field as many times as needed.
    • A help button with help text is available for each field.
    • Groups of fields, or field sets, may be accessed separately (employing distinct window tabs or expand/collapse regions for each field set.)
  • The MAW metadata editor contrasts to UFDC/SobekCM metadata editor:
    • New or revised MAW metadata templates will be uploadable via the website by users with metadata template uploading permission.
    • Uploaded metadata templates will automatically generate new metadata item editor interface forms that web users can select and use to edit metadata.
    • MAW will provide a bulk change editor interface, and more user-specifiable metadata fields will be supported.
    • Independent of UFDC/SobekCM, MAW operates its own internal UF Library website, so MAW updates will be more agile and easier to manage.
    • MAW will directly output or transform edited metadata for download or transmission, for example, to HathiTrust.
    • MAW will be able transform metadata to METS format and load it to the SobekCM/UFDC Builder at designated times, for example when the SobekCM builder is typically not bogged down with heavy traffic, to avoid slowdowns during the day.
  • Many other capable database and metadata editors exist, such as Oxygen, that run on standalone PCs, however since this metadata editor runs on the MAW hardware and software platform, it will take advantage of:
    • internal UF Library web access by users (so no need for independent installations and updates of software packages on staff PCs)
    • existing UF-proprietary xml2rdb technology with relational database functionality
    • metadata editor users can use forms and so do not have to think like programmers, and
    • Easy integration with xml2rdb technology and the MAW platform, which already has coding to prepare metadata METS loads into UFDC, based on prior work.
